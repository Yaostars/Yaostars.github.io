2. Orders and Functions #

Linux orders often looks like:

command [-options] [parameter]

ls #

ls [-a -l -h] [path]
-a: show all (including hidden) files and folders
-h: make information easier to read (often used together with “-l”)
-l: list detailed information about files and folders 
ls -alh /	# show detailed information neatly of all files and folders in root directory

cd & pwd & mkdir #

cd: enter to path directory
pwd: show current work path
mkdir: create a new folder
 
cd bob
pwd
mkdir [bob/]new_folder

touch & cat & more #

touch: create a new file
cat: show the content of a file
more: see the content of a file and support page turning 
touch [bob/]test.txt
cat [bob/]test.txt
more [bob/]test.txt

cp & rm & mv #

cp [-r] old_file new_file	# Choose -r when copying a folder
rm [-r -f] file1 file2...	# -f means force delete, -r means deleting folders
mv old_path new_path
cp: copy a file or folder
mv: move a file or folder to a new path
rm: delete files or folders (use a wildcard “*” to achieve fuzzy matching) 
rm -f *.txt	# Delete all files ending in ".txt"

which & find #

which order_name
find begin_path -name "file_name"
find begin_path -size +|-n[kMG]	# "+" means larger, "-" means smaller, k/M/G are size units
which: find the path of order files
find: find files or directories location, can be used together with wildcard “*” 
find / -name test*	# find paths of all files and directories starting in "test" since the root directory
find / -size - 100M	# find paths of all files and directories smaller than 100M since the root directory

grep & wc #

grep [-n] keyword file_path	# choose "-n" when you want to see the the line number in the file where the keyword is located
wc [-c -m -l -w] file_path
grep: find the keyword location in the file, can be used in conjunction with other commands, connected by pipeline symbols “|” in between
wc: count the number of lines, words, etc. in the file
 -c: count number of bytes
 -m: count number of characters
 -l: count number of lines
 -w: count number of words 
cat test.txt | grep itheima | grep itcast	# find sentences containing "itheima" or "itcast" in test.txt
wc -c test.txt

echo & tail #

echo content
tail [-f -num] path	# choose "-f" when consistently follow a file, choose "-num" to point the number of last lines you want to see in the files (default with 10 lines)
echo: print content instantly (suggest enclosing the output content with double quotation marks), can be used together with “>” or “>>” to write the output content to the file on the right. It can also be used with “`” to show the result of a command.
 >: overwrite
 >>: add write
tail: show the last lines in a file 
echo `pwd` >> test.txt	# append the result of the "pwd" command to the test.txt
tail -f -20 test.txt	# continuously check the last 20 lines content of test.txt

vi / vim #

vim is an upgraded version of vi, which fully includes all the command methods of vi.

In command mode:
 i: enter input mode at the cursor position
 a: enter input mode after the cursor position
 I: enter input mode at beginning of the line that the cursor position
 A: enter input mode at the end of the line that the cursor position
 o: enter input mode at the next line of cursor position
 O: enter input mode at the above line of cursor position
 esc: back to command mode in any situation
 [n]dd: delete n lines from cursor position
 [n]yy: copy the n+1 lines from current line
 p: paste
 u: revoke
 ctrl+r: reverse revoke
 gg: cursor jump to the first line
 G: cursor jump to the end of current line
 dG: delete all from the current line
 dgg: delete all above from the current line
 d$: delete content till the end of current line
 d0: delete content till the head of current line

In baseline command mode:
 :wq: save and quit
 :q: quit without save
 :q!: forced quit
 :w: save
 :set nu: show the number of current line
 :set paste: set the paste mode
